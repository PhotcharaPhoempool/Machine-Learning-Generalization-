--------------------------------------------------------------------------------
0) ภาพรวมโจทย์
--------------------------------------------------------------------------------
ฟังก์ชันเป้าหมายจริง : f(x) = sin(pi * x)
ข้อมูล               : สุ่ม x จำนวน n จุด จากการแจกแจงเอกรูป U[-1, 1]
                       แล้วบวก noise แบบปรกติ ~ N(0, sigma^2)   =>  y = f(x) + noise
แบบจำลอง (โจทย์)     : 2 แบบ คือ constant (ค่าคงที่)  และ  linear (เชิงเส้น)

ตัวประมาณค่า E_out ที่ศึกษา 3 แบบ :
  1) resubstitution           - เทรนบนข้อมูลทั้งหมด n จุด แล้ววัด error บน n จุดเดิม
  2) holdout                  - แบ่งข้อมูลเป็น train/test เทรนบน train วัด error บน test
  3) k-fold cross-validation  - แบ่งเป็น k ก้อน วนให้แต่ละก้อนเป็น test แล้วเฉลี่ย error

สิ่งที่ต้องหา :
  (a) ประมาณ E_out ด้วยทั้ง 3 วิธี บนข้อมูล "ชุดเดียว" แล้วเทียบกับ E_out จริง
  (b) ทำซ้ำหลายชุดข้อมูล หา bias, variance, MSE ของแต่ละวิธี เทียบว่าใครเอนเอียง/แปรปรวน
  (c) เปลี่ยนสัดส่วนการแบ่งของ holdout และจำนวน fold k เทียบ variance ว่าแบบใดแม่นกว่า
  (d) ปรับ n และ sigma แล้วดูผลต่อ bias และ variance ของแต่ละวิธี

1) แนวคิดสำคัญที่ใช้
--------------------------------------------------------------------------------
* E_out จริงของโมเดลที่เทรนแล้ว (นิยามที่ใช้ในงานนี้ = test error จริง รวม noise):
      E_out(D) = E_x[ (g_D(x) - f(x))^2 ]  +  sigma^2
  - พจน์แรก  = ความคลาดเคลื่อนเฉลี่ยของโมเดลเทียบฟังก์ชันจริง (ประมาณด้วยกริดถี่ 1001 จุด)
  - พจน์ sigma^2 = ส่วนของ noise ที่ลดไม่ได้ (irreducible error) ที่จุดทดสอบใหม่ก็มี noise
  - ใช้โมเดลที่ "เทรนบนข้อมูลเต็มชุด" เป็นตัวอ้างอิง (เพราะเป็นโมเดลที่จะเอาไปใช้จริง)

* bias / variance / MSE ของ "ตัวประมาณ" (คำนวณจากการทำซ้ำ M ชุดข้อมูล):
      bias      = ค่าเฉลี่ยของค่าประมาณ  -  ค่าเฉลี่ยของ E_out จริง
      variance  = ความแปรปรวนของค่าประมาณ (มันเหวี่ยงมากแค่ไหนในแต่ละชุดข้อมูล)
      MSE       = ค่าเฉลี่ยของ (ค่าประมาณ - E_out จริง)^2   โดยประมาณเท่ากับ  bias^2 + variance
  * bias เป็นบวก = ประมาณสูงเกินจริง / เป็นลบ = ประมาณต่ำเกินจริง (มองโลกในแง่ดี)

4) ผลลัพธ์และการตีความ (งานหลัก : constant + linear)
--------------------------------------------------------------------------------
(a) บนข้อมูลชุดเดียว:
    - resub มักให้ค่าต่ำกว่า E_out จริง (มองโลกในแง่ดี) เพราะวัด error บนข้อมูลที่เทรนไปแล้ว
    - holdout และ k-fold ให้ค่าใกล้ E_out จริงกว่า เพราะวัดบนข้อมูลที่โมเดลไม่เคยเห็น

(b) ทำซ้ำหลายชุด (สรุปทิศทางที่ได้):
    - resub  : bias เป็น "ลบ" (optimistic) แต่ variance ต่ำ
    - holdout: bias เกือบศูนย์ (แทบไม่เอนเอียง) แต่ variance "สูงสุด" (เพราะ test มีไม่กี่จุด)
    - k-fold : bias เกือบศูนย์ และ variance ต่ำกว่า holdout มาก -> สมดุลดีที่สุด
    - ตรวจความถูกต้อง: ทุกแถว MSE ~= bias^2 + variance (สอดคล้องกับสูตร)

(c) เปลี่ยนการแบ่ง:
    - holdout: test set ใหญ่ขึ้น (ratio มากขึ้น) -> variance ของค่าประมาณลดลง
               แต่ถ้า train เหลือน้อยเกินไป bias จะเริ่มโต (โมเดลเทรนจากข้อมูลน้อย)
    - k-fold : k มากขึ้น -> variance ลดลง และเข้าใกล้ LOOCV (k=n)
    - สรุป: k-fold (โดยเฉพาะ k=5 หรือ 10) ให้ค่าประมาณที่ "แม่นยำ/นิ่ง" กว่า holdout

(d) ปรับ n และ sigma:
    - n มากขึ้น   -> variance ของทุกวิธีลดลง (ข้อมูลเยอะ ประมาณนิ่งขึ้น)
    - sigma มากขึ้น -> variance เพิ่มขึ้น และ resub ยิ่งมองโลกในแง่ดีชัดขึ้น (bias ติดลบมากขึ้น)

5) ผลลัพธ์ส่วนศึกษาเพิ่มเติม (poly3) และข้อสังเกตสำคัญ
--------------------------------------------------------------------------------
เพิ่มโมเดล poly3 (ยืดหยุ่นสูง) เพื่อดูว่าตัวประมาณมีพฤติกรรมอย่างไรเมื่อโมเดลซับซ้อน:
    - resub ของ poly3 ยิ่งมองโลกในแง่ดี (ช่องว่างระหว่างค่าประมาณกับ E_out จริงกว้างขึ้น)
    - variance ของค่าประมาณ poly3 สูงกว่า linear แทบทุกกรณี
    - เมื่อ n เพียงพอ (เช่น n=40) poly3 ก็ประมาณได้แคบรอบ E_out จริง

** ข้อสังเกต (ไม่ใช่ข้อผิดพลาด) **
    ที่ n น้อย (เช่น n=10) variance ของ poly3 "พุ่งสูงมหาศาล" (หลักร้อย-หลักหมื่น)
    เพราะพหุนามดีกรี 3 fit บนจุดฝึกไม่กี่จุด แล้วเหวี่ยงหลุดกรอบบนช่วง [-1,1]
    = อาการ overfitting สุดขั้ว ซึ่งเป็นบทเรียนที่ต้องการให้เห็นพอดี
    ในกราฟจึงใช้แกน y แบบ log เพื่อให้เทียบ linear กับ poly3 พร้อมกันได้

6) สรุปบทเรียน
--------------------------------------------------------------------------------
- resubstitution   : เอนเอียงต่ำ (มองโลกในแง่ดี) ห้ามใช้ประเมินสมรรถนะจริง
- holdout          : ไม่เอนเอียงมาก แต่ "แปรปรวนสูง" ขึ้นกับว่าแบ่งอย่างไร/สุ่มได้ชุดไหน
- k-fold CV        : ไม่เอนเอียงมาก และ "แปรปรวนต่ำ" กว่า holdout -> เป็นตัวเลือกที่ดีที่สุด
- ยิ่งข้อมูลน้อย/noise มาก/โมเดลยืดหยุ่นสูง -> การประมาณ E_out ยิ่งยากและแปรปรวนมากขึ้น
